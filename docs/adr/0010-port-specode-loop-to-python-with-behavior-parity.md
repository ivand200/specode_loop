# Port Specode Loop to Python with Behavior Parity

Specode Loop will add a separate `scripts/specode_loop.py` Python implementation while preserving the current command-line contract, logging behavior, sandbox lifecycle, bundled workflow skill sync, and success sentinel handling. The existing shell implementation remains side-by-side during the port, with the Python implementation intended to replace it after behavior parity is proven. The Python version uses only the standard library so the rewrite stays compact and easy to compare against existing behavior. The blessed Python invocation is `uv run python scripts/specode_loop.py ...`; the Python file does not need to be directly executable. A minimal `pyproject.toml` may define pytest as the only development dependency so the Python port can be verified with `uv run pytest`, and README should document the Python command without removing the current shell usage yet. Runtime behavior should not depend on third-party packages. This revises the earlier "narrow shell runner" implementation choice without changing the product boundary: Specode Loop remains a narrow Docker Sandbox runner, and the Python port should first be judged against the existing regression behavior rather than used as a design revision.
